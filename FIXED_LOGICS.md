# FIXED_LOGICS.md - 構造設計大原則および確定計算ロジック仕様書

本仕様書は、本アプリケーションにおける構造設計の「最重要原則」および「確定計算ロジック」を明文化し、プログラム全域での完全な物理的・力学的整合性を保証するマスター仕様書である。

---

## 1. 構造マスター大原則 (最重要項目)

1. **通り芯（グリッド）の絶対性**:
   - 斜め軸を除き、全ての通り芯（グリッド）は完全な垂直線（90°：X座標固定、Y軸に平行）および水平線（0°/180°：Y座標固定、X軸に平行）である。
   - グリッドの配置は、標準モジュール（455mm）に縛られず、**「1mm 単位」**で正確かつ自由に定義される。
   
2. **柱の配置制約**:
   - すべての柱は、必ずこれら垂直・水平通り芯の **「交点（厳密なミリメートル整数座標）」** にのみ配置される。
   
3. **耐力壁の配置制約**:
   - すべての耐力壁は、必ず通り芯に厳密に沿って **「柱と柱の間（グリッド交点間）」** にのみ設置される。
   
4. **基礎梁の配置制約**:
   - すべての基礎梁は、必ず通り芯に厳密に沿って **「グリッド交点（柱）」** を結ぶように配置される。
   
5. **スラブ端点の配置制約**:
   - すべての基礎スラブの頂点（端点）は、必ず垂直・水平通り芯 of **「交点」** に厳密に一致（スナップ）して定義される。

## 2. 確定計算・荷重同期ロジック仕様

### ① スラブの三角形割増しロジック
スラブの荷重を各辺に分配する際、形状が幾何学的に「三角形」である場合には、等分布荷重への換算において割増係数（デフォルト `1.33`）を適用する。

- **三角形の場合**:
  $$B = \frac{\text{Area} \times \text{triangleMultiplier}}{\text{edgeLen}}$$
- **三角形以外（矩形・台形）の場合**:
  $$B = \frac{\text{Area}}{\text{edgeLen}}$$

*(※ Area はポリゴン面積、edgeLen は対応する辺 of 長さ。単位はメートル)*

### ② 隣り合うスラブの梁への荷重同期・按分ロジック (複数スラブまたがり含む)
基礎梁スパンに対し、取り付いている隣接スラブの辺との重複関係および比率に基づいて、トータルの等分布荷重 $w$ (kN/m) を合算・按分する。

1. **サイドごとの集計（右側 `side 1` / 左側 `side -1`）**:
   梁スパン区間とスラブの辺の区間が重複している長さ（重複比率 $Ratio$）に基づき、スラブ辺の負担幅 $B$ と接地圧 $\sigma$ を按分する。
   - **右側 (`side 1`)**: 平均負担幅 $B_{\text{side1}}$、代表接地圧 $\sigma_{\text{side1}}$
   - **左側 (`side -1`)**: 平均負担幅 $B_{\text{side2}}$、代表接地圧 $\sigma_{\text{side2}}$

2. **両サイドの合算・加重平均**:
   - **全負担幅 $B_{\text{total}}$ (単純加算)**:
     $$B_{\text{total}} = B_{\text{side1}} + B_{\text{side2}}$$
   - **最終接地圧 $\sigma_{\text{final}}$ (負担幅を重みとする加重平均)**:
     $$\sigma_{\text{final}} = \frac{\sigma_{\text{side1}} \cdot B_{\text{side1}} + \sigma_{\text{side2}} \cdot B_{\text{side2}}}{B_{\text{total}}}$$
     *(※ $B_{\text{total}} = 0$ の場合は、設計条件の地耐力 $f_e$ またはシステム基準の接地圧を適用)*

### ③ 長期応力計算ロジック
同期された等分布荷重 $w$ (kN/m) に基づき、スパンごとの長期モーメント $M$ および長期せん断力 $Q$ を求める。

- **等分布荷重 $w$**:
  $$w = \sigma_{\text{final}} \times B_{\text{total}} + w_{\text{self}}$$
  *(※ $w_{\text{self}}$ は基礎梁の自重 kN/m)*

- **中央曲げモーメント $M_{\text{mid}}$**:
  $$M_{\text{mid}} = \frac{w \cdot L^2}{8}$$

- **端部曲げモーメント $M_{\text{end}}$**:
  $$M_{\text{end}} = \frac{w \cdot L^2}{12}$$

- **長期せん断力 $Q_L$**:
  $$Q_L = \frac{w \cdot L}{2}$$

*(※ $L$ は柱間（支点間）のスパン長（メートル）)*

## 3. 壁量計算・N値計算ロジック仕様

### ① 必要壁量算定（見下げ面積・見上げ面積）の算出ロジック
地震力に対する必要壁量の算出に使用する床面積は、「建築基準法（見下げ方式）」と「住宅性能表示（見上げ方式）」で計算が異なる。

1. **基準法（見下げ面積方式）**:
   床面積 $A_{\text{floor}}$ に、小屋裏収納（平均天井高 $H \le 1.4\text{m}$）の床面積 $A_{\text{attic}}$ に、小屋裏天井高補正係数（$1.4 / 2.1 \approx 0.67$）を掛け了床面積を単純加算する。
   $$A_{\text{seismic}} = A_{\text{floor}} + \left( A_{\text{attic}} \times 0.67 \right)$$

2. **性能表示（見上げ面積方式 - 吹き抜け/バルコニー投影）**:
   見上げ投影面積を含めるため、吹き抜け $A_{\text{void}}$、ポーチ・屋根 $A_{\text{porch}}$、およびバルコニー床面積 $A_{\text{balcony}}$（40%低減）を考慮して加算する。
   - **1階 (1F)**:
     $$A_{\text{seismic, 1F}} = A_{\text{floor, 1F}} + \left( A_{\text{attic, 1F}} \times 0.67 \right) + \left( A_{\text{attic, 2F}} \times 0.67 \right) + A_{\text{porch}} + \left( A_{\text{balcony, 1F}} \times 0.4 \right)$$
   - **2階 (2F)**:
     $$A_{\text{seismic, 2F}} = A_{\text{floor, 2F}} + \left( A_{\text{attic, 2F}} \times 0.67 \right) + A_{\text{void}}$$

### ② 斜め耐力壁のXY方向換算（壁倍率分解）
通り芯方向に対して角度 $\theta$ を持った「斜め壁」の耐力壁長 $L$、倍率 $\alpha$ は、X軸およびY軸のそれぞれ独立した成分へと余弦・正弦定理に基づいて分解・換算される。

- **X軸方向の有効壁長 $W_{\text{eff, X}}$**:
  $$W_{\text{eff, X}} = L \times \cos \theta \times \alpha$$
- **Y軸方向の有効壁長 $W_{\text{eff, Y}}$**:
  $$W_{\text{eff, Y}} = L \times \sin \theta \times \alpha$$

### ③ 柱引抜N値計算ロジック
柱の接合金物（ホールダウン金物等）を選定するための柱頭・柱脚の付加引抜力算定ロジック（N値計算）の確定。

- **N値算定基本公式**:
  $$N = \max\left( 0,\ \left( \alpha \times B \times K \right) + N_{\text{transfer}} - L \right)$$

1. **柱隣接壁倍率差 $\alpha$（引抜力モーメント成分）**:
   柱に隣接する対向方向（右/左、または上/下）の有効壁倍率の差に、筋交い等の取り付き方による Table 3-14 補正係数 $corr$ を加えた実効値。
   $$\alpha = \left| \sum \alpha_{\text{Right}} - \sum \alpha_{\text{Left}} \right| + corr$$

2. **柱位置低減係数 $B$**:
   - **出隅柱（角柱）の場合**:
     $$B = 0.8$$
   - **中間柱の場合**:
     $$B = 0.5$$

3. **階高補正係数 $K$**:
   実効階高 $H$ が $3.2\text{m}$ を超える場合には、引抜モーメントが大きくなるため階高補正を施す。
   - **$H \le 3.2\text{m}$ の場合**:
     $$K = 1.0$$
   - **$H > 3.2\text{m}$ の場合**:
     $$K = \frac{H}{2.7}$$

4. **上階からの引抜力伝達 $N_{\text{transfer}}$**:
   - **2階 (2F) 柱**: $N_{\text{transfer}} = 0$
   - **1階 (1F) 柱**: 2階の真上に存在する柱の引抜モーメント伝達分。
     $$N_{\text{transfer}} = \alpha_{2F} \times B_{2F} \times K_{2F}$$

5. **押さえ込み荷重 $L$（鉛直長期荷重による相殺効果）**:
   - **簡易法**:
     - 1F中間柱 (2F重なりなし): $L = 0.6$
     - 1F角柱 (2F重なりなし): $L = 0.4$
     - 1F中間柱 (2F重なりあり): $L = 1.6$
     - 1F角柱 (2F重なりあり): $L = 1.0$
     - 2F中間柱: $L = 0.6$
     - 2F角柱: $L = 0.4$
   - **詳細法（支配面積による長期実荷重換算）**:
     $$L = \frac{\left( w_{\text{roof}} \cdot A_{\text{tributary}} \right) + \left( w_{\text{floor}} \cdot A_{\text{tributary}} \right)}{5.3}$$

## 4. 基礎モード表示精度仕様（ミリメートル・小数点第3位規則）

基礎連動解析モードの視認性および厳密な物理量の不整合を防ぐため、UI画面および計算結果レポート上の数値の小数点丸め処理は以下の規則を徹底する。

- **小数点以下第3位（`toFixed(3)`）まで表示する対象**:
  1. **面積**: スラブ個別面積（$\text{m}^2$）、基礎梁支配・負担面積（$\text{m}^2$）
  2. **長さ**: 柱間スパン（$\text{m}$）、基礎梁全長（$\text{m}$）、スラブ各辺の長さ（$\text{m}$）、スパンごとの負担幅 $B$（$\text{m}$）
  3. **計算値**: 設計用等分布荷重 $w$（$\text{kN/m}$）、長期モーメント $M$（$\text{kN}\cdot\text{m}$）、長期せん断力 $Q$（$\text{kN}$）、接地圧 $\sigma$（$\text{kN/m}^2$）、検定比（判定結果）

- **本精度表示規則から除外する対象（整数・定格表示）**:
  1. **鉄筋径**: D10, D13, D16 などの呼び名、および定格断面積
  2. **配筋・鉄筋本数**: 配筋ピッチ（200mm等、整数表示）や本数（整数表示）

## 5. 通り芯（グリッド）の追加・オフセット・履歴管理ロジック

通り芯の動的な追加および削除、ならびにそれらの操作の履歴同期に関する確定ロジック。

### ① グリッド追加（オフセット）の基準点選定ロジック
ユーザーがキャンバス上をクリックして通り芯を追加する際、基準となる座標（$Base$）は以下の優先順位で決定される。
1. **既存通り芯へのスナップ**:
   クリック位置から 100px 以内に既存の通り芯（$Grid$）が存在する場合、その通り芯の座標を $Base$ とする。
2. **クリック位置の採用**:
   近傍に通り芯が存在しない場合、クリック位置の座標を標準モジュール（455mm）にスナップさせた値を $Base$ とする。
   $$Base = \text{round}\left( \frac{\text{ClickPos}}{\text{Module}} \right) \times \text{Module}$$

### ② 手動追加グリッドの優先順位（ブラックリスト回避）
DXF解析等により自動生成された通り芯が「削除（Blacklist）」状態にある場合でも、ユーザーが明示的に手動追加（`manualGrid`）した座標は、削除リスト（`deletedGrid`）を上書きして優先的に表示・計算対象とする。
- **解析時ロジック**:
  $$Grids_{final} = \left( Grids_{auto} \cup Grids_{manual} \right) - \left( Grids_{deleted} - Grids_{manual} \right)$$

### ③ 履歴（Undo/Redo）同期ロジック
グリッドの状態変更（追加・削除・名称変更）は、柱や壁のデータと同様に `historyStack` に完全なスナップショットとして保存される。
- **保存対象**: `manualGridX/Y`, `deletedGridX/Y`, `gridXNames/YNames`
- **復元**: 履歴からの復元時、`GridEngine.analyzeGrids()` を即座に実行し、最新の物理座標配列（`gridXCoords/YCoords`）を再生成する。

## 6. 4分割法（壁の釣り合い）計算ロジック

建物各階の四分割図における側端部の必要壁量と存在壁量の算定ロジック。

### ① 側端部領域の定義（境界線）
建物の外郭（境界ポリゴン）の最小・最大座標に基づき、各方向の 1/4 領域を定義する。
- `xLineL = minX + (W / 4)`
- `xLineR = maxX - (W / 4)`
- `yLineT = maxY - (H / 4)`
- `yLineB = minY + (H / 4)`

### ② 側端部必要壁量の面積算定（AreaEngine）
側端部面積（$Area_{side}$）は、建物構成ポリゴンを上記の境界線でクリッピングした面積の合計とする。
- **X方向・上側（XT）**: `yLineT` から `maxY` までの範囲
- **X方向・下側（XB）**: `minY` から `yLineB` までの範囲
- **Y方向・左側（YL）**: `minX` から `xLineL` までの範囲
- **Y方向・右側（YR）**: `xLineR` から `maxX` までの範囲

### ③ 側端部存在壁量の集計（StructuralEngine）
各側端部領域に重心（中心点）が含まれる耐力壁の有効壁長を合計する。
- **X方向（水平壁）**: `vxt`, `vxb` に集計
- **Y方向（垂直壁）**: `vyl`, `vyr` に集計

### ④ 判定ロジック
1. **充足率の算出**: $Sufficiency = Existence / (Area_{side} \times q)$
2. **壁率比の算出**: $Ratio = \min(Sufficiency1, Sufficiency2) / \max(Sufficiency1, Sufficiency2)$
3. **合格判定**: $(Sufficiency1 \ge 1.0 \text{ かつ } Sufficiency2 \ge 1.0)$ または $Ratio \ge 0.5$ の場合に **OK** と判定する。
