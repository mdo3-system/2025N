# Git・バージョン管理厳守プロトコル

## 1. 変更の絶対原則
いかなる理由があっても、以下の手順を踏まずに既存のコード（.py, .js, .php, .dxf関連等）を書き換えてはならない。

## 2. 変更実行のワークフロー (Check-Before-Write)
変更の指示を受けた際、AIエージェントは以下の手順を順に実行し、チャットで報告すること。

1. **現状確認 (Status Check)**
   - 現在のブランチ名と、最後にFIXしたバージョン（例：v1.16）を確認する。
2. **バックアップ作成 (Pre-Change Commit)**
   - 変更を加える直前の状態を必ずコミット、または「backup/v1.16_before_fix」のようなブランチを作成し、退避させる。
3. **バージョン更新の提案**
   - 変更後の新しいバージョン番号（例：v1.16.1 や v1.17-alpha）を提示し、ユーザーの承認を得る。
4. **インクリメンタルな変更**
   - ファイルを丸ごと書き換えるのではなく、修正が必要な関数やブロックのみを特定して変更する。
5. **完了後の確定 (Post-Change Commit)**
   - 変更が完了したら、新バージョン名をコミットメッセージに含めてコミットする。

## 3. 禁止事項
- **一括削除の禁止:** `rm` コマンドや、コードの大部分を削除する行為は、バックアップ完了のログを提示するまで禁止する。
- **FIX済みロジックの保護:** `// [FIXED]` タグがある箇所や、構造計算の核心ロジックについては、変更が必要な場合、必ず「なぜ変更が必要か」の論理的説明を行い、許可を得ること。
